AgentSearchBench: A Benchmark for AI Agent Search in the Wild

AgentSearch · SIGIR 2026 · Melbourne | Naarm, Australia · July 24, 2026

Paper Webpage GitHub

Overview

The AgentSearchBench challenge is organized as part of the AgentSearch workshop at SIGIR 2026.

Agent ecosystems are essential infrastructure for deploying AI systems on complex, real-world tasks. Identifying the right agent for a given task largely depends on understanding agent capabilities, which are often compositional, execution-dependent, and difficult to assess from textual descriptions alone. However, collecting reliable agent relevance signals at scale is costly and technically demanding. Consequently, existing benchmarks rely on well-specified functionalities, controlled candidate pools, or purely executable task queries, leaving realistic agent search scenarios insufficiently studied.

The AgentSearchBench challenge aims to explore agent search in the wild by evaluating retrieval and reranking methods against execution-grounded performance signals. Recent studies have shown that semantic similarity alone is insufficient for identifying high-performing agents. Nevertheless, it remains unclear which retrieval approaches best capture true agent utility, which ranking signals most reliably predict execution performance, how lightweight behavioral probing compares to description-based methods, whether execution-aware signals can close the gap between semantic similarity and actual agent performance, and how well current methods generalise across diverse task types and agent providers.

This challenge will investigate these questions across nearly 10,000 real-world agents drawn from multiple providers. The collected data and evaluation framework will be released as a public package to support agent discovery and selection research in the broader AI and information retrieval communities.

Dataset Introduction

AgentBase — the Agent Repository

The benchmark is built from 9,759 real-world agents sourced from public platforms. Rather than synthetic agents, AgentBase captures practical challenges such as overlapping capabilities, inconsistent documentation, and heterogeneous description formats.

Agents are collected from three real-world platforms, ensuring broad domain coverage and authentic capability diversity:

  • GPT Store – General-purpose GPTs
  • Google Cloud Marketplace – Enterprise agents
  • AgentAI Platform – Specialised agents

Each agent is normalised into a unified four-part schema: agent metadata, capability description, usage guidance, and availability & constraints.

AgentSearch Challenge

Develop systems that retrieve and rank AI agents from a large repository given task specifications. The challenge covers both executable queries and high-level task descriptions, with evaluation grounded in agent execution performance.

Challenge Overview

The challenge consists of three tracks targeting different levels of task complexity:

  • Track 1 – Single-Agent Task Query: Rank agents for concrete executable instructions, evaluated via execution outputs scored by an LLM.
  • Track 2 – Multi-Agent Task Query: Rank agents for composed tasks with multiple subtasks; relevance depends on coverage across all subtasks.
  • Track 3 – High-Level Task Description: Rank agents for abstract goals evaluated across multiple executable queries.

Dataset & Tasks

  • 9,759 real-world agents with heterogeneous descriptions and capabilities
  • 2,452 single-agent queries and 500 multi-agent queries
  • 259 high-level task descriptions with associated executable queries
  • Execution-based relevance labels derived from 66,740 runs

Evaluation

Evaluation is grounded in real agent execution. A 5-point LLM-as-judge assesses output quality, with human validation confirming high agreement (Cohen’s κ = 0.93). Metrics include NDCG, Precision, Recall, MRR, and completeness-based measures.

Submissions

Submission details will be released with the challenge. Participants will submit ranked lists of agents per query using a standard IR-style format. Teams may submit multiple runs, and reproducibility submissions are encouraged.

Submission Format

    # query_id  0  agent_id  relevance_score
    q001        0  agt:gpt:2518c1    3
    q001        0  agt:gcp:ab3d12    1
    q002        0  agt:gpt:ff91ca    2

Relevance Scale

  • [3] Perfectly relevant — exact match and correct output
  • [2] Highly relevant — mostly correct but partial
  • [1] Related — relevant but does not solve the task
  • [0] Irrelevant — not related to the task

Frequently Asked Questions

Are there restrictions on which models I can use?

No. You can use any model, including proprietary, open-weight, or fine-tuned systems.

Can I modify the prompts?

Yes. Any prompting strategy is allowed, including zero-shot, few-shot, or fine-tuned approaches.

How many submissions are allowed?

There is no limit. Multiple submissions and reproducibility runs are encouraged.

What is the goal of the challenge?

The goal is to develop effective retrieval and ranking methods for AI agents, including prompt design, model selection, and fine-tuning strategies.

How is relevance validated?

Relevance is based on execution results scored by an LLM-as-judge, with strong agreement validated by human annotators.

Will winners be announced?

Yes. Top-performing teams will be announced at COLM 2026.
